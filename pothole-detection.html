Capabilities
Key Features
▶On-Device Real-Time Detection

YOLOv8n model exported to ONNX and runs entirely on-device — no cloud, no latency. Achieves ~41ms inference on CPU hardware for true real-time detection at highway speeds.

▶GPS Coordinate Logging

Every detected pothole is tagged with GPS coordinates and logged to a local database, enabling road hazard mapping and crowd-sourced road condition reporting.

▶Voice Alert System

Hands-free audio alerts fire when a pothole is detected ahead, keeping the driver's eyes on the road. Alert timing is tuned to give adequate reaction time at typical driving speeds.

▶82.2% mAP Accuracy

Trained on a custom dataset of road surface imagery. Achieves 82.2% mean average precision — balancing detection sensitivity with false-positive suppression for practical deployment.

▶Dual Deployment: Android + PWA

Ships as a native Android app (Kotlin) for full hardware access, plus a Progressive Web App for zero-install browser-based access — same model, two delivery targets.

How It Works
Technical Implementation
[01]
Model TrainingYOLOv8n fine-tuned on custom pothole dataset. Data augmentation: flips, mosaic, HSV jitter to improve generalization across lighting and road surface conditions.
[02]
ONNX Export & OptimizationModel exported to ONNX format for cross-platform inference. Quantization-aware export keeps model size minimal while preserving accuracy for mobile hardware.
[03]
Android Integration (Kotlin)ONNX Runtime Android library runs inference on each camera frame. Camera2 API provides low-latency frame capture. Detection results overlaid via custom Canvas renderer.
[04]
GPS & Alert PipelineAndroid Location API fuses GPS + network positioning. Detected pothole events trigger TextToSpeech alerts and write to SQLite with timestamp + coordinates.
[05]
PWA DeploymentWeb version uses ONNX.js in the browser for client-side inference. Service Worker enables offline capability and installable app experience on any device.
