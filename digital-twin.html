Capabilities
Key Features
▶UR5e Pick-and-Place Workcell

Universal Robots UR5e arm with Robotiq 2F-85 gripper simulated in Webots. Picks color-coded boxes from a moving conveyor belt and sorts them to designated bins — replicating real Amazon warehouse operations.

▶Live ML Anomaly Detection

Isolation Forest model runs in real-time on robot sensor data — joint torques, cycle times, gripper force. Flags statistical anomalies that indicate equipment wear, jams, or unexpected behavior.

▶Real-Time SCADA Dashboard

Live web dashboard streams telemetry via ROS 2 rosbridge WebSocket — pick counts, anomaly scores, cycle times, uptime, and conveyor speed updated in real-time using Chart.js visualizations.

▶Full Docker Containerization

Entire system — ROS 2, Webots, ML pipeline, and dashboard — runs in Docker containers. One command brings up the complete simulation environment on any machine.

▶Amazon-Style Warehouse Logic

Implements real warehouse sorting logic: color detection → bin assignment → pick scheduling → conveyor pacing. Models the control architecture used in actual fulfillment center automation.

How It Works
Technical Implementation
[01]
Simulation Layer (Webots)Webots world models UR5e kinematics, Robotiq gripper physics, conveyor dynamics, and box spawning. ROS 2 controllers interface via the ros2_control framework.
[02]
Robot Controller (ROS 2)MoveIt 2 plans pick-and-place trajectories. Color detection node identifies box type from camera feed. State machine orchestrates the full pick-sort-place cycle.
[03]
Anomaly Detection (scikit-learn)Isolation Forest trained on nominal operation data. Real-time inference on ROS 2 topic callbacks — each sensor reading scored and published to the anomaly topic.
[04]
WebSocket Bridgerosbridge_server exposes all ROS 2 topics as WebSocket streams. Dashboard subscribes to telemetry topics and renders live charts without any polling or REST calls.
[05]
Dockerized DeploymentMulti-container Docker Compose setup: ros2-sim, ml-node, dashboard-server. Shared ROS_DOMAIN_ID enables inter-container topic communication.
