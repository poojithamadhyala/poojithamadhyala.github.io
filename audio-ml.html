Capabilities
Key Features
▶MFCC Feature Extraction

Mel-Frequency Cepstral Coefficients capture the perceptually-relevant spectral shape of audio events. 13 coefficients per frame encode enough information for accurate event classification while keeping the feature vector compact.

▶CNN Audio Classifier

Convolutional neural network processes the 2D MFCC spectrogram. Conv layers learn frequency-temporal patterns distinctive to each audio event class — more robust to background noise than simpler approaches.

▶ONNX Export for Edge

Trained PyTorch model exported to ONNX format. ONNX Runtime enables inference on edge hardware without PyTorch dependency — dramatically reducing deployment footprint for embedded systems.

▶Low-Latency Inference

Optimized preprocessing pipeline minimizes per-frame overhead. ONNX Runtime session reuse eliminates initialization cost on repeated inference — enabling real-time classification at target throughput.

▶End-to-End ML Pipeline

Complete pipeline from raw audio to classification: data loading, augmentation, MFCC extraction, CNN training, evaluation, and ONNX export — all reproducible from a single codebase.

How It Works
Technical Implementation
[01]
Audio PreprocessingRaw waveforms resampled to 22050 Hz. Framed into overlapping windows with 50% hop. Hann window applied before FFT to reduce spectral leakage.
[02]
MFCC ExtractionFFT magnitude fed through 40-band Mel filterbank. Log-Mel energies transformed via DCT to 13 MFCCs per frame. Delta and delta-delta coefficients appended for temporal dynamics.
[03]
CNN Architecture4-layer CNN: Conv2D → BatchNorm → ReLU → MaxPool repeated twice, followed by global average pooling and fully-connected classifier head. Dropout for regularization.
[04]
Training PipelinePyTorch training loop with Adam optimizer, cross-entropy loss, cosine LR schedule. Data augmentation: time-stretch, pitch-shift, additive noise for robustness.
[05]
ONNX Deploymenttorch.onnx.export with dynamic axes for variable-length input. ONNX Runtime inference session with graph optimizations enabled — ready for Raspberry Pi, Jetson, or any edge target.
